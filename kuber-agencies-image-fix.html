① Root Cause Analysis
HTTP 404 · File Not Found
File Does Not Exist on Server
The web server found no file at the requested path. This is the most common cause.
  • Image was never uploaded to the server.
  • Wrong directory — uploaded to /uploads/ but URL points to /images/.
  • Case mismatch: Linux is case-sensitive. Cement.webp ≠ cement.webp.
  • Filename typo: underscore vs hyphen, extra spaces, unicode characters.
  • Extension mismatch: file saved as .jpeg but URL requests .jpg.
  • Symlink pointing to a path that no longer exists.
HTTP 403 · Permission Denied
File Exists but Server Can't Read It
File is on disk but the web server process (www-data / nginx) lacks read permission.
  • File uploaded with permissions 600 (owner-only). Needs 644.
  • Parent directory missing execute bit. Needs 755 — without it the server can't traverse into it.
  • File owned by root or a deploy user, not www-data.
  • SELinux or AppArmor policy blocking access (less common on shared hosting).
  • Apache Options -Indexes combined with missing index — returns 403 for directories, not files, but often confused.
CORS / Network Error
Missing Cross-Origin Header
Browser blocks the resource because the server didn't send Access-Control-Allow-Origin.
  • JS fetch() or an external page loading your images triggers CORS policy.
  • Apache mod_headers not enabled (a2enmod headers not run).
  • .htaccess Header directives ignored because AllowOverride None in vhost config.
  • Nginx missing add_header Access-Control-Allow-Origin * in the location block.
  • CDN or reverse proxy stripping your CORS headers before they reach the browser.
HTTP 200 but Broken
Wrong MIME Type or Corrupt File
File is delivered but browser refuses to render it as an image.
  • Server returns text/plain or application/octet-stream for .webp — browser ignores it.
  • WebP MIME type not registered: older Apache/Nginx without explicit image/webp mapping.
  • File was corrupted during SFTP transfer (ASCII mode used instead of binary).
  • File is actually a renamed PNG/JPG — extension is .webp but header is wrong.
  • Truncated upload: partial file due to a failed transfer.
  • PHP/CMS accidentally serving a 200 HTML error page instead of the binary image.
② Step-by-Step Server Fix

Server-Side Fix — Apache & Nginx

Run every step in order over SSH. Fixes 404, 403, CORS, and MIME errors permanently.

A
Verify Files Exist & Fix Naming
Check each category directory. All filenames must be lowercase with hyphens. No spaces, no uppercase, no underscores.
# List what's actually on disk in each category ls -la /var/www/html/images/cement/ ls -la /var/www/html/images/steel/ ls -la /var/www/html/images/paints/ # Check a specific file you expect to exist ls -la /var/www/html/images/cement/ultratech-opc53-cement-bags-chennai.webp ls -la /var/www/html/images/cement/ultratech-opc53-cement-bags-chennai.jpg # Find any uppercase filenames (these will 404 on Linux!) find /var/www/html/images/ -name "*[A-Z]*" # Rename all to lowercase in one shot (safe — skips already-lowercase) find /var/www/html/images/ -depth -name "*[A-Z]*" | while read f; do target=$(dirname "$f")/$(basename "$f" | tr 'A-Z' 'a-z') [ "$f" != "$target" ] && mv "$f" "$target" && echo "Renamed: $f → $target" done # Replace underscores with hyphens (URL convention) find /var/www/html/images/ -depth -name "*_*" | while read f; do target=$(echo "$f" | tr '_' '-') [ "$f" != "$target" ] && mv "$f" "$target" && echo "Renamed: $f → $target" done
B
Fix File Permissions & Ownership
Files need 644, directories need 755. Owner must be www-data (Apache/Nginx web user on Ubuntu/Debian). On CentOS/RHEL use apache:apache instead.
# Fix ALL image file permissions recursively find /var/www/html/images/ -type f -exec chmod 644 {} \; # Fix ALL directory permissions recursively find /var/www/html/images/ -type d -exec chmod 755 {} \; # Set correct ownership (Ubuntu/Debian) chown -R www-data:www-data /var/www/html/images/ # --- If on CentOS / RHEL / Fedora, use this instead: # chown -R apache:apache /var/www/html/images/ # Verify the fix worked (should show -rw-r--r-- and drwxr-xr-x) ls -la /var/www/html/images/ ls -la /var/www/html/images/cement/
C
Fix Apache — .htaccess Config
Create or update /var/www/html/.htaccess. This fixes WebP MIME type, CORS headers, and cache headers in one file. First enable required modules.
# Enable the required Apache modules (run as root) a2enmod headers a2enmod mime a2enmod expires # Restart Apache to apply systemctl restart apache2 # Confirm modules are active apache2ctl -M | grep -E 'headers|mime|expires'
# Create/overwrite /var/www/html/.htaccess with this content: cat > /var/www/html/.htaccess << 'EOF' # ── WebP MIME type ───────────────────────────────────────── AddType image/webp .webp # ── CORS headers for images (required for JS fetch / CDN) ── <FilesMatch "\.(webp|jpg|jpeg|png|gif|svg)$"> Header set Access-Control-Allow-Origin "*" Header set Access-Control-Allow-Methods "GET, HEAD, OPTIONS" Header set Vary "Accept" </FilesMatch> # ── Long-term cache headers (1 year) ─────────────────────── <IfModule mod_expires.c> ExpiresActive On ExpiresByType image/webp "access plus 1 year" ExpiresByType image/jpeg "access plus 1 year" ExpiresByType image/png "access plus 1 year" ExpiresByType image/gif "access plus 1 year" ExpiresByType image/svg+xml "access plus 1 year" </IfModule> <FilesMatch "\.(webp|jpg|jpeg|png|gif|svg)$"> Header set Cache-Control "max-age=31536000, public, immutable" </FilesMatch> # ── Security: hide server version in error pages ─────────── ServerSignature Off EOF # Verify it was written correctly cat /var/www/html/.htaccess # Test Apache config — must show Syntax OK before restarting apachectl configtest # Reload (not restart) — zero downtime systemctl reload apache2
# IMPORTANT: .htaccess only works if AllowOverride is NOT "None" # Check your virtual host config: grep -r "AllowOverride" /etc/apache2/sites-enabled/ # If you see "AllowOverride None", change it to "AllowOverride All": # Edit the file shown (usually 000-default.conf or your domain.conf) nano /etc/apache2/sites-enabled/000-default.conf # Find this block and change None → All: # <Directory /var/www/html> # AllowOverride All ← change from None to All # </Directory> # After editing, test and reload: apachectl configtest && systemctl reload apache2
D
Fix Nginx Config (if using Nginx)
Add the WebP MIME type and CORS/cache headers to your server block. Edit /etc/nginx/sites-available/kuberagencies.in (or your domain config file).
# Open your Nginx site config nano /etc/nginx/sites-available/kuberagencies.in # ── Add this INSIDE your server { } block ──────────────────────── # WebP MIME type (add to the types{} block or use include) types { image/webp webp; image/jpeg jpg jpeg; image/png png; image/gif gif; image/svg+xml svg svgz; } # Image-specific location block — CORS + cache headers location ~* \.(webp|jpg|jpeg|png|gif|svg)$ { add_header Access-Control-Allow-Origin "*" always; add_header Access-Control-Allow-Methods "GET, HEAD, OPTIONS" always; add_header Vary "Accept" always; add_header Cache-Control "max-age=31536000, public, immutable" always; expires 1y; # Allow OPTIONS preflight if ($request_method = OPTIONS) { add_header Access-Control-Allow-Origin "*"; add_header Access-Control-Allow-Methods "GET, HEAD, OPTIONS"; add_header Content-Length 0; return 204; } } # ───────────────────────────────────────────────────────────────── # Test config — must say "syntax is ok" and "test is successful" nginx -t # Reload (zero downtime) systemctl reload nginx
E
Generate Missing WebP Files from JPGs
If you only have JPG originals on the server, batch-convert them to WebP. Quality 85 gives excellent compression with no visible loss.
# Install cwebp (Google's WebP encoder) sudo apt update && sudo apt install -y webp # Single file conversion cwebp /var/www/html/images/cement/ultratech-opc53-cement-bags-chennai.jpg \ -o /var/www/html/images/cement/ultratech-opc53-cement-bags-chennai.webp \ -q 85 # Bulk convert ALL .jpg files in the entire images tree # Skips files that already have a .webp counterpart find /var/www/html/images/ -name "*.jpg" | while read jpg; do webp="${jpg%.jpg}.webp" if [ ! -f "$webp" ]; then cwebp "$jpg" -o "$webp" -q 85 && echo "✓ Converted: $webp" else echo "↷ Skipped (exists): $webp" fi done # Fix permissions on the newly created .webp files find /var/www/html/images/ -name "*.webp" -exec chmod 644 {} \; chown -R www-data:www-data /var/www/html/images/ # Verify conversion: compare file sizes ls -lh /var/www/html/images/cement/
F
Verify MIME Types Are Actually Served
After applying the config, confirm the server really sends image/webp. Do this from your SSH session.
# Check Content-Type header returned by the server curl -sI https://kuberagencies.in/images/cement/ultratech-opc53-cement-bags-chennai.webp \ | grep -i "content-type" # Expected: content-type: image/webp # Check CORS header is present curl -sI https://kuberagencies.in/images/cement/ultratech-opc53-cement-bags-chennai.webp \ | grep -i "access-control" # Expected: access-control-allow-origin: * # Full header dump for one URL curl -sv https://kuberagencies.in/images/cement/ultratech-opc53-cement-bags-chennai.webp \ -o /dev/null 2>&1 | grep "<" # Check HTTP status codes for all three example URLs for url in \ "https://kuberagencies.in/images/cement/ultratech-opc53-cement-bags-chennai.webp" \ "https://kuberagencies.in/images/steel/jsw-tmt-fe500d-bars-chennai.webp" \ "https://kuberagencies.in/images/paints/asian-paints-apex-exterior-chennai.webp"; do code=$(curl -o /dev/null -s -w "%{http_code}" "$url") echo "$code $url" done
③ Bulk Fix & Audit Script

image-fix.sh — Save & Run on Your Server

This script does everything in one shot: detects missing files, fixes permissions/ownership, converts JPGs to WebP, and produces a full audit report. Save as image-fix.sh, make executable, run as root via SSH.

-d "$IMAGES_DIR" ]; then log "${RED}✗ ERROR: $IMAGES_DIR does not exist!${RESET}" log " Run: mkdir -p $IMAGES_DIR/cement $IMAGES_DIR/steel $IMAGES_DIR/paints" exit 1 fi log "${GREEN}✓ Images directory found: $IMAGES_DIR${RESET}" # ── STEP 2: Rename uppercase/underscore filenames ─────────── log "\n${BOLD}[2/6] Normalizing filenames (lowercase + hyphens)...${RESET}" find "$IMAGES_DIR" -depth -name "*[A-Z]*" | while read f; do target=$(dirname "$f")/$(basename "$f" | tr 'A-Z' 'a-z') if [ "$f" != "$target" ]; then mv "$f" "$target" log " ${YELLOW}Renamed:${RESET} $(basename $f) → $(basename $target)" fi done find "$IMAGES_DIR" -depth -name "*_*" | while read f; do target=$(echo "$f" | tr '_' '-') if [ "$f" != "$target" ]; then mv "$f" "$target" log " ${YELLOW}Renamed:${RESET} $(basename $f) → $(basename $target)" fi done log "${GREEN}✓ Filename normalization complete${RESET}" # ── STEP 3: Fix permissions & ownership ─────────────────── log "\n${BOLD}[3/6] Fixing permissions and ownership...${RESET}" find "$IMAGES_DIR" -type f | while read f; do perms=$(stat -c "%a" "$f") if [ "$perms" != "644" ]; then chmod 644 "$f" FIXED_PERMS=$((FIXED_PERMS + 1)) log " ${YELLOW}Fixed perms:${RESET} $f ($perms → 644)" fi done find "$IMAGES_DIR" -type d | while read d; do perms=$(stat -c "%a" "$d") if [ "$perms" != "755" ]; then chmod 755 "$d" log " ${YELLOW}Fixed dir perms:${RESET} $d ($perms → 755)" fi done chown -R "$WEB_USER:$WEB_USER" "$IMAGES_DIR" log "${GREEN}✓ Ownership set to $WEB_USER:$WEB_USER${RESET}" # ── STEP 4: Convert JPGs to WebP where WebP is missing ────── log "\n${BOLD}[4/6] Converting JPG → WebP (missing only)...${RESET}" if command -v cwebp &> /dev/null; then find "$IMAGES_DIR" -name "*.jpg" -o -name "*.jpeg" | while read jpg; do webp="${jpg%.*}.webp" if [ ! -f "$webp" ]; then if cwebp "$jpg" -o "$webp" -q "$WEBP_QUALITY" -quiet; then chmod 644 "$webp" chown "$WEB_USER:$WEB_USER" "$webp" CONVERTED=$((CONVERTED + 1)) log " ${GREEN}Converted:${RESET} $(basename $webp)" else log " ${RED}✗ Failed to convert:${RESET} $jpg" fi fi done else log " ${YELLOW}⚠ cwebp not found. Install: sudo apt install webp${RESET}" fi log "${GREEN}✓ WebP conversion done${RESET}" # ── STEP 5: Detect missing/orphaned images ────────────────── log "\n${BOLD}[5/6] Scanning for missing .webp AND .jpg pairs...${RESET}" find "$IMAGES_DIR" -name "*.jpg" | while read jpg; do webp="${jpg%.jpg}.webp" [ ! -f "$webp" ] && log " ${YELLOW}⚠ No WebP pair:${RESET} $jpg" && MISSING=$((MISSING+1)) done find "$IMAGES_DIR" -name "*.webp" | while read webp; do jpg="${webp%.webp}.jpg" [ ! -f "$jpg" ] && log " ${YELLOW}⚠ No JPG pair:${RESET} $webp" && MISSING=$((MISSING+1)) done # ── STEP 6: HTTP status audit ─────────────────────────────── log "\n${BOLD}[6/6] HTTP audit (checking each URL returns 200)...${RESET}" if command -v curl &> /dev/null; then find "$IMAGES_DIR" -type f \( -name "*.webp" -o -name "*.jpg" \) | while read f; do url="$BASE_URL${f#/var/www/html}" code=$(curl -o /dev/null -s -w "%{http_code}" --max-time 5 "$url") if [ "$code" = "200" ]; then log " ${GREEN}✓ 200${RESET} $url" else log " ${RED}✗ $code${RESET} $url" BROKEN_URLS=$((BROKEN_URLS + 1)) fi done else log " ${YELLOW}⚠ curl not found — skipping HTTP audit${RESET}" fi # ── SUMMARY ───────────────────────────────────────────────── log "\n${CYAN}═══════════════════════════════════════════${RESET}" log "${BOLD} SUMMARY${RESET}" log "${CYAN}═══════════════════════════════════════════${RESET}" log " Finished: $(date)" log " Full log: $LOG_FILE" log "" log " Next: Apply Apache (.htaccess) or Nginx config" log " to fix CORS headers and WebP MIME type." log "${CYAN}═══════════════════════════════════════════${RESET}" # ── HOW TO RUN ─────────────────────────────────────────────── # Save this file as image-fix.sh, then: # chmod +x image-fix.sh # sudo bash image-fix.sh

⑤ Debugging Workflow

How to Test Each Fix

Use these steps in order to isolate exactly which layer is failing — DNS, server, permissions, MIME, or CORS.

1
Direct URL Access in Browser
Open the image URL directly in a new browser tab. If it shows a broken icon or blank → 404/403. If it renders → CORS (a script is blocked, not direct access). If it prompts download → MIME type issue.
https://kuberagencies.in/images/cement/ultratech-opc53-cement-bags-chennai.webp
2
curl — Exact HTTP Status & Headers
Run from your SSH session or local machine. This gives you the raw HTTP status code and all response headers with zero ambiguity.
# Get HTTP status code only curl -o /dev/null -s -w "HTTP %{http_code}\n" \ https://kuberagencies.in/images/cement/ultratech-opc53-cement-bags-chennai.webp # Full response headers (look for Content-Type and Access-Control-Allow-Origin) curl -sI https://kuberagencies.in/images/cement/ultratech-opc53-cement-bags-chennai.webp # Test with an explicit Origin header to simulate CORS preflight curl -sI -H "Origin: https://kuberagencies.in" \ https://kuberagencies.in/images/cement/ultratech-opc53-cement-bags-chennai.webp \ | grep -i "access-control" # Batch status check for all three failing URLs for url in \ "https://kuberagencies.in/images/cement/ultratech-opc53-cement-bags-chennai.webp" \ "https://kuberagencies.in/images/steel/jsw-tmt-fe500d-bars-chennai.webp" \ "https://kuberagencies.in/images/paints/asian-paints-apex-exterior-chennai.webp"; do echo "$(curl -o /dev/null -s -w '%{http_code}' $url) $url" done
3
Browser DevTools — Network Tab
Open DevTools (F12), go to the Network tab, filter by Img. Reload the page. Click any red/failed request to see exact status, headers, and error type. Look for content-type in the Response Headers column.
# What to look for in Network tab: Status 404 → File missing on disk Status 403 → Permission error (chmod/chown needed) Status 200 → Check Response Headers: Content-Type must be image/webp (failed) → CORS error — check Console tab for exact message # Console tab CORS error will look like: Access to fetch '...' from origin '...' has been blocked by CORS policy: No 'Access-Control-Allow-Origin' header is present on the requested resource.
4
Verify File Integrity (Not Corrupt)
If curl returns 200 but the image still renders broken, the file itself is likely corrupt or was transferred in ASCII mode. Use file to check the real format.
# Check what the file actually is (regardless of extension) file /var/www/html/images/cement/ultratech-opc53-cement-bags-chennai.webp # Should say: RIFF...WebP image data # If it says "ASCII text" or "HTML document" → corrupted upload # Check file size (0 bytes = empty/failed upload) ls -lh /var/www/html/images/cement/ultratech-opc53-cement-bags-chennai.webp # Check the first few bytes (WebP magic bytes: RIFF + WEBP) xxd /var/www/html/images/cement/ultratech-opc53-cement-bags-chennai.webp | head -2 # Expected: 52 49 46 46 ... 57 45 42 50 (RIFF....WEBP)
5
Check Apache / Nginx Error Logs
Error logs give the exact server-side reason for every failed request. Always check these when curl doesn't give enough information.
# Apache error log — last 50 lines, live follow tail -f /var/log/apache2/error.log # Apache access log — see every request with status code tail -f /var/log/apache2/access.log | grep "/images/" # Nginx error log tail -f /var/log/nginx/error.log # Nginx access log filtered to images only tail -f /var/log/nginx/access.log | grep "/images/" # Search for 403 and 404 errors in last 200 lines tail -200 /var/log/apache2/access.log | awk '$9 ~ /^(403|404)$/'

⑥ Prevention Strategy
📁
Naming Convention
Enforce a strict naming rule before upload. Any deviation causes a 404 on Linux.
  • Always lowercase
  • Hyphens only — no spaces, underscores, or special chars
  • Format: brand-product-variant-city.webp
  • Example: ultratech-opc53-cement-bags-chennai.webp
  • Always upload both .webp AND .jpg pairs
🔼
Upload Checklist
Before going live, run this checklist for every image:
  • File uploaded via SFTP in binary mode (not ASCII)
  • chmod 644 applied immediately after upload
  • chown www-data:www-data set on the file
  • Direct URL opens correctly in browser tab
  • curl -I <url> returns 200 + image/webp
  • WebP counterpart exists for every JPG
🔄
Deployment Pipeline
Add automated validation to prevent broken images reaching production:
  • Add the image-fix.sh audit script to your deploy hook
  • Block deployment if any image URL returns non-200
  • Auto-convert JPGs to WebP as a post-deploy step
  • Auto-set permissions with find ... -exec chmod
  • Notify via Slack/email if broken images are detected
🗜️
Image Optimisation
Reduce file size before upload to improve load speed on mobile:
  • Target size: 50–150 KB per product image
  • WebP at quality 80–85 (use cwebp -q 85)
  • Resize to max 800×600px — cards display at 400×210
  • Strip EXIF metadata: exiftool -all= *.jpg
  • Use jpegoptim --max=85 *.jpg for JPG originals
🌐
CDN Usage (Optional but Recommended)
A CDN eliminates CORS issues, adds global caching, and handles WebP auto-conversion:
  • Cloudflare (free) — proxy your domain, enable Polish for auto-WebP
  • Cloudflare automatically adds CORS headers and sets correct MIME types
  • Images cached at edge nodes globally — faster loads in India
  • Enable Browser Cache TTL: 1 year in Cloudflare dashboard
  • Use Cloudflare Images for managed uploads with auto-resize
🔍
Ongoing Monitoring
Set up monitoring so broken images alert you before customers see them:
  • Schedule image-fix.sh as a cron job — run daily at 2 AM
  • Use UptimeRobot (free) to monitor key image URLs
  • Add Google Search Console — it flags broken images in GSC reports
  • Set ENABLE_IMG_DIAGNOSTICS = true in dev, false in prod
  • Add a <noscript> fallback img for SEO crawlers
